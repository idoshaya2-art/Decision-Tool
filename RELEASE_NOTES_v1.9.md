# Release Notes — v1.9 Group Governance

גרסה זו משלימה את רצף הביקורת: מידע מאושר הופך למספר מבוקר, המספר מוקרן ב־Digital Twin, מחקרי השוק מזינים את ההחלטה, מנוע האופטימיזציה בוחר סל פעולות שלם, והקבוצה מאשרת אותו בתהליך מתועד.

## v1.7 — Market Intelligence

- קישור תוצאות MR להחלטות לפי רבעון, אזור, מוצר ורמה.
- הצגת התוצאה המדויקת, המקור, רמת הביטחון והמשמעות העסקית.
- Value of Information למחקר שעשוי להפוך החלטה.
- הפרדה קשיחה בין Actual, מחקר, אומדן והנחה.

## v1.8 — Integrated Q9 Portfolio Optimizer

- בחינת סלי פעולות אפשריים תחת מגבלת תקציב ורצפת מזומן.
- אכיפת חוקים, תלויות, התנגשויות וזמני השפעה לפני דירוג.
- סימולציית Low/Base/High עד Q9 לכל סל.
- דירוג 50% ביצועים כספיים ו־50% פוטנציאל עתידי, בניכוי סיכון.
- רגישות משקלים ו־Pareto frontier למניעת דיוק מדומה.

## v1.9 — Group Governance

- ישיבת החלטה הנשענת על ריצת האופטימיזציה שנשמרה.
- חמש פרסונות: כספים, שיווק, תפעול, אסטרטגיה ויו״ר.
- בקרות מכונה חוסמות על חוקיות, תקציב, ראיות, תלויות ותזמון.
- כלל אישור: ארבעה תומכים לפחות, יו״ר תומך וללא וטו מתפקיד חוסם.
- שמירת נימוקים, התנגדויות ועמדות מיעוט בגיבוי ובשחזור.
- אין שליחה אוטומטית למערכת INTOPIA.

## שדרוג ענן

ממערכת v1.6 יש להריץ פעם אחת ובסדר זה:

```text
supabase/migration_v1.7_market_intelligence.sql
supabase/migration_v1.9_group_governance.sql
```

לאחר מכן מעלים את הקוד ל־GitHub ומבצעים Deploy latest commit ב־Render. אין להעלות `.env`, מפתחות API, דוחות אמיתיים או גיבויים.
