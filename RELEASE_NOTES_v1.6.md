# EMBA TAU Simulation AI Decision OS — v1.6

## Digital Twin רבעוני

גרסה זו מחליפה תחזית מצרפית בלבד במודל מצב רבעוני. המודל מתחיל מ־Actual מאושר ונעול, מעתיק אותו לתרחיש ומחשב את השפעת הפעולות מ־Q4 ועד Q9 בלי לשנות את נתוני האמת.

### יכולות חדשות

- Timeline רבעוני של מזומן, חוב, חייבים, מלאי, קיבולת, מפעלים ורמות טכנולוגיה.
- Low / Base / High לכל חבילת פעולות.
- זמני השפעה מפורשים לייצור, מכירה, גבייה, מפעלים, רישיונות והעברות רכיבים.
- העברות כספים בין אזורים עם עדכון מקור ויעד ושמירת המזומן המאוחד.
- שמירת Snapshots נעולים וריצות תרחיש ב־Supabase.
- שילוב התוצאה בסימולטור, בחבילת ההחלטות וב־Decision Agent.
- הכללת הישויות החדשות בגיבוי ובשחזור.

### שדרוג מסד קיים

לאחר הורדת גיבוי מלא, יש להריץ פעם אחת ב־Supabase SQL Editor:

```text
supabase/migration_v1.6_digital_twin.sql
```

לאחר העלאת הקוד ל־GitHub ופריסה ב־Render, בדיקת הבריאות צריכה להציג:

```json
{"status":"ok","version":"1.6.0-digital-twin","backend":"supabase"}
```

בדיקת המודל:

```text
GET /api/digital-twin/Q4
```

התגובה צריכה לכלול `baseline`, את `locked=true`, ואת רשימת `runs` האחרונה.
