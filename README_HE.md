# EMBA TAU Simulation — v0.5 Decision Intelligence

פלטפורמת ענן תומכת החלטה לסימולציית INTOPIA. המערכת מתחילה את עבודת התכנון בתחילת Q4, לאחר קליטה נפרדת של תוצאות Q1, Q2 ו־Q3, ומסתכלת תמיד קדימה עד Q9.

## מה חדש בגרסה

- ממשק הנהלה פיננסי חדש ותפריט המבורגר סגור כברירת מחדל.
- קליטת CSV, Excel, PDF טקסטואלי וקובצי טקסט; כל חילוץ מוצג לאישור לפני שהוא נכנס לנתוני האמת.
- דוח רבעוני ודוח מצטבר לפי כל הרבעונים שאושרו.
- מצב כספי מאוחד ולפי מדינה, כולל מזומן, חוב, הון חוזר, מלאי והתחייבויות.
- אומדן ניהולי שקוף: 50% ביצועי עבר ו־50% פוטנציאל עתידי. זה אינו הציון הרשמי של המשחק.
- תחזית Rolling ל־Q9, זיהוי מגמות, המלצות תמחור, קיבולת, מו״פ ומחקרים רלוונטיים.
- מעבדת תרחישים עם Low / Base / High, סדר פעולות, תקציב, רזרבת מזומן ובדיקת היתכנות.
- Decision Agent אופציונלי של OpenAI. הוא קורא רק נתונים מאושרים דרך כלי קריאה ואינו משנה נתונים.
- שמירת נתונים ב־Supabase PostgreSQL וקבצים ב־Supabase Storage; אין SQLite ואין הסתמכות על הדיסק הזמני של Render.
- גיבוי ושחזור ZIP מלאים, כולל נתונים, קבצים, imports, מחקרים, תרחישים ושיחות Agent.

## מסלול העבודה

1. העלו מסמך אסטרטגיה ויעדי Q9.
2. העלו בנפרד את פלט Q1, את פלט Q2 ואת פלט Q3.
3. עברו על תצוגת החילוץ ולחצו **אישור קליטה** רק לאחר בדיקה.
4. השלימו רק נתונים שהמערכת לא הצליחה לחלץ, בעיקר מידע איכותני.
5. פתחו את **חדר ההחלטות Q4**: מצב כספי, ציון פנימי, תחזית Q9 והמלצות.
6. העבירו פעולות אפשריות למעבדת התרחישים ובדקו אותן מול התקציב ורצפת המזומן.
7. בכל סוף רבעון העלו את הפלט החדש, אשרו אותו והורידו גיבוי.

- `DEPLOYMENT_CHECKLIST_HE.md` — פריסה ושדרוג ב־Supabase, GitHub ו־Render.
- `MIGRATION_v0.4_to_v0.5_HE.md` — שדרוג מערכת v0.4 קיימת בלי למחוק נתונים.

## משתני הסביבה שחייבים להזין ב־Render

| משתנה | מה מזינים |
|---|---|
| `SUPABASE_URL` | כתובת Project URL של Supabase |
| `SUPABASE_SECRET_KEY` | Secret key או `service_role` של השרת בלבד |
| `APP_ACCESS_PASSWORD` | סיסמת צוות חדשה וחזקה |

ה־Blueprint כבר מגדיר `INTOPIA_BACKEND=supabase`, את ה־bucket הפרטי `intopia-files`, הגנת סיסמה ומגבלות קבצים.

## הפעלת Decision Agent — אופציונלי

הוסיפו ב־Render רק אם רוצים להפעיל את הצ׳אט:

```text
OPENAI_AGENT_ENABLED=true
OPENAI_API_KEY=<מפתח שרת של פרויקט OpenAI>
OPENAI_MODEL=<שם מודל הזמין בפרויקט ה-API שלכם>
```

המפתח נשמר ב־Render בלבד. אין לשים אותו ב־GitHub, בקובץ JavaScript, בצילום מסך או בצ׳אט. שימוש ב־OpenAI API עשוי להיות כרוך בתשלום נפרד; יתר האפליקציה פועלת גם כאשר ה־Agent כבוי.

## מגבלות חילוץ קבצים

- CSV ו־Excel עם כותרות ברורות: חילוץ אוטומטי + מסך אישור.
- PDF המכיל טקסט: חילוץ מדדים בסיסיים + מסך אישור.
- PDF סרוק ותמונה: הקובץ נשמר, אך מסומן כדרוש OCR/בדיקה. המערכת אינה ממציאה מספרים שלא זוהו.
- את מיפוי דוחות המשחק המדויק יש לכייל מול דוגמה אמיתית מכל סוג דוח.

## בדיקות

```text
python -m pip install -r requirements-dev.txt
python -m pytest -q
```

בדיקת Supabase אמיתית מופעלת רק עם `RUN_SUPABASE_INTEGRATION=1`. לעולם אין לבצע commit של `.env`, מפתחות, דוחות אמיתיים או קובצי גיבוי.
