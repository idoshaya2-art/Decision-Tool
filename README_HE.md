## מה חדש ב־v1.5 — Evidence-to-Number Gate

- כל מספר מהותי בהמלצה עובר ביקורת ראיות לפני שהוא מוצג כפעולה לביצוע.
- לכל טענה מספרית נשמרת שרשרת שקופה: `מקור → Actual או הנחה → נוסחה → טווח → תוצאה → המלצה`.
- הטענות מסווגות ל־`מאושר`, `מותנה` או `חסום`, כולל מקור, נוסחה, טווח Low/Base/High, רמת ביטחון והמידע החסר.
- לחיצה על **איך הגענו למספרים?** בכרטיס החלטה פותחת את כל שרשרת החישוב והסתירות שנמצאו.
- מספר חסום אינו ניתן לאימוץ מתוך חדר ההחלטות ואינו יכול להיכנס לחבילת החלטות מוכנה.
- חבילת החלטות מקבלת סטטוס `ready_for_team_review`, `needs_evidence` או `blocked_evidence` לפי איכות הראיות, בנוסף לבדיקות חוק, תקציב ותזמון.
- ה־Decision Agent קיבל כלי `get_evidence_gate` וחייב לבדוק אותו לפני הצגת פעולה מספרית או יצירת טיוטת החלטה.
- ביקורות הראיות נשמרות במסד, נכללות בגיבוי ובשחזור וניתנות לשליפה דרך API.
- שדרוג מסד קיים דורש הרצה חד־פעמית של `supabase/migration_v1.5_evidence_gate.sql`.
- 58 בדיקות אוטומטיות עוברות ובדיקה אחת של Supabase אמיתי מדולגת ללא פרטי ענן.

## מסלול העבודה

1. העלו מסמך אסטרטגיה ויעדי Q9.
2. בחרו **זיהוי רבעון אוטומטי** והעלו בנפרד את פלט Q1, את פלט Q2 ואת פלט Q3.
3. עברו על תצוגת החילוץ ולחצו **אישור קליטה** רק לאחר בדיקה.
4. עבור מסמך חוקים חדש, עברו למרכז החוקים ואשרו, דחו או השאירו לבדיקה כל מועמד. אישור מייעד אותו לגרסת Rulebook הבאה ואינו משנה את הגרסה הפעילה מיד.
5. השלימו רק נתונים שהמערכת לא הצליחה לחלץ, בעיקר מידע איכותני.
6. פתחו את **חדר ההחלטות Q4**: מצב כספי, ציון פנימי, תחזית Q9 והמלצות.
7. העבירו פעולות אפשריות למעבדת התרחישים ובדקו אותן מול התקציב ורצפת המזומן.
8. לפני סיום החלטות הרבעון ודאו שתחזית הרבעון הבא נעולה במסך **תובנות ומגמות**.
9. בכל סוף רבעון העלו את הפלט החדש, אשרו אותו, עברו על תחקיר Forecast→Actual ואשרו או דחו הצעות כיול.
10. הורידו גיבוי מלא לאחר סיום התחקיר.

## הפעלת Decision Agent

הוסיפו ב־Render רק אם רוצים להפעיל את הצ׳אט:

```text
OPENAI_AGENT_ENABLED=true
OPENAI_API_KEY=<מפתח שרת של פרויקט OpenAI>
OPENAI_MODEL=gpt-5.6-sol
OPENAI_MAX_OUTPUT_TOKENS=2400
```

ה־Blueprint כבר מגדיר `OPENAI_AGENT_ENABLED=true`, את `OPENAI_MODEL=gpt-5.6-sol` ואת מגבלת הפלט; בפועל נדרש להזין בעצמכם רק את `OPENAI_API_KEY`. המפתח נשמר ב־Render בלבד. אין לשים אותו ב־GitHub, בקובץ JavaScript, בצילום מסך או בצ׳אט. שימוש ב־OpenAI API כרוך בחשבון API ובמכסה נפרדים מ־ChatGPT. המסך `/api/agent/status` מציג בדיוק איזה משתנה חסר בלי להציג את ערכו.

## מגבלות חילוץ קבצים

- CSV ו־Excel עם כותרות ברורות: חילוץ אוטומטי + מסך אישור.
- PDF המכיל טקסט: חילוץ מדדים בסיסיים + מסך אישור.
- PDF סרוק ותמונה: הקובץ נשמר, אך מסומן כדרוש OCR/בדיקה. המערכת אינה ממציאה מספרים שלא זוהו.
- פלט INTOPIA במבנה שנבדק על Q1–Q3: מפענח מדויק ובדיקות התאמה.
- שינוי עתידי במבנה גיליונות המשחק יסומן לבדיקה ולא יומר למספרים מומצאים.
