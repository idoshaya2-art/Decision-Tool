# INTOPIA DSS — גרסה 0.3 Clean

פלטפורמת עבודה ריקה לסימולציית INTOPIA. לא נכללים בה נתוני חברה, תוצאות רבעון, אסטרטגיה, יעדים, תרחישים, מחירים או הנחות לדוגמה.

## מה נשאר במערכת

נשמרו רק רכיבים מבניים שאינם נתוני החברה:

- Q1–Q9, אזורים, מוצרים ודגמים.
- כללי התאמת X–Y.
- אפשרויות ההלוואה הקבועות במערכת.
- קטלוג מחקרי השוק.
- מנועי החישוב, התמחור, התרחישים והייצוא.

## מסלול ההקמה

בכניסה הראשונה מרכז השליטה מציג רשימת הקמה:

1. הגדרת שם החברה.
2. העלאת אסטרטגיה ראשונית.
3. העלאת יעדי Q9.
4. העלאת Datalog / כללי המשחק.
5. העלאת Q1–Q3 יחד או דוח רבעוני.
6. אישור נתונים מובנים לרבעון.

המדדים וצמתי ההחלטה לא מציגים אפסים או התראות מלאכותיות לפני שיש נתונים אמיתיים.

## סוגי קבצים שניתן להעלות

- אסטרטגיה ראשונית.
- יעדי Q9.
- Datalog / כללי משחק.
- Q1–Q3 משולב.
- דוח רבעוני.
- מחקר שוק.
- Gazette.
- הסכם / עסקה.
- PDF, Excel, CSV, תמונות ומסמכים נוספים.

הקבצים נשמרים במערכת כמקורות אמת. בגרסה זו הקבצים עדיין אינם מפוענחים אוטומטית לשדות; מנגנון המיפוי והפענוח הוא שלב הפיתוח הבא.

## עבודה עם נתונים מובנים

במסך **רבעון** ניתן:

- להזין נתוני מטה ופיננסים.
- להוסיף שורה בודדת של אזור–מוצר–דגם.
- ליצור בלחיצה שלד רבעוני ריק של כל האזורים, המוצרים והדגמים.
- לשמור רק לאחר השלמת אזור, מוצר ודגם.

מסכי התמחור והתרחישים מתחילים ריקים ואינם כוללים הנחות לדוגמה.

## הפעלה ב-Windows

1. חלצו את קובץ ה-ZIP.
2. ודאו שמותקן Python 3.11 ומעלה.
3. לחצו פעמיים על `run_app.bat`.
4. פתחו בדפדפן `http://127.0.0.1:8000`.

## הפעלה ב-Mac / Linux

```bash
chmod +x run_app.sh
./run_app.sh
```

## שמירת נתונים

- מסד הנתונים: `data/intopia.db`
- קבצים שהועלו: `data/uploads/`

החבילה המסופקת אינה מכילה מסד נתונים קיים. הוא נוצר ריק בהפעלה הראשונה.

## איפוס עתידי

למחיקת כל נתוני החברה והקבצים שהועלו:

- Windows: `reset_data.bat`
- Mac / Linux: `./reset_data.sh`

האיפוס דורש הקלדת `RESET` כדי למנוע מחיקה מקרית.

- בדיקת API, העלאת קובץ, הגדרות והזנת פעילות.
- בדיקות מנוע החישוב הקיימות.
