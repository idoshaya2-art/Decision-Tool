# שדרוג v0.5 ל־v0.6 בלי מחיקת נתונים

1. באפליקציה הפעילה הורידו **גיבוי מלא** ושמרו אותו במחשב.
2. ב־Supabase פתחו SQL Editor והפעילו פעם אחת את `supabase/migration_v0.6.sql`.
3. ודאו שהפקודה הסתיימה בהצלחה. היא מוסיפה עמודת שער מטבע בלבד ואינה מוחקת רשומות.
4. העלו ל־GitHub את כל קובצי v0.6, כך שהם יחליפו קבצים בעלי אותו שם ויוסיפו קבצים חדשים.
5. ב־Render → Environment הזינו בעצמכם `OPENAI_API_KEY` ו־`OPENAI_MODEL`; ודאו `OPENAI_AGENT_ENABLED=true`.
6. בצעו Deploy ובדקו `/api/health` ולאחריו `/api/agent/status`.
7. העלו את Q1, Q2 ו־Q3 במצב זיהוי אוטומטי, בדקו את תצוגת החילוץ ואשרו כל רבעון.

אין להעלות ל־GitHub דוחות אמת, גיבויים, `.env` או מפתחות API.
