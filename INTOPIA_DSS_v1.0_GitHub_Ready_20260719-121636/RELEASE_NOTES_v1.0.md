# Release Notes — v1.0 AI Decision OS

## תוצאה

הגרסה הופכת את המערכת מדשבורד נתונים למערכת תומכת החלטה מבוססת חוקים, תקציב, תחזית וראיות.

## נוסף

- Rulebook v1.0 עם יותר מ־50 חוקים ו־19 הגדרות טופס החלטה.
- מקורות וגרסאות, סתירות פתוחות ומרכז חיפוש.
- אכיפת חוק על דוחות, פעולות, תרחישים וחבילות החלטה.
- בדיקות תקציב, רצפת מזומן, קיבולת, תזמון, X–Y, תמחור, מו״פ, הובלה, רישוי ומחקרים.
- חבילת החלטות הניתנת לייצוא JSON ורק לבדיקת הצוות.
- AI tools לחיפוש חוק, בדיקת פעולה ויצירת טיוטת חבילה.
- אינדקס קטעי מקור למסמכים שהועלו, חיפוש שלהם מתוך צ׳אט ה־AI וניתוח מועמדי חוקים למסמכי Setup.
- תור אישור אנושי למועמד חדש, הבהרה או סתירה; אף מועמד אינו משנה את Rulebook הפעיל אוטומטית.
- Audit להרצות AI, תחזיות, אופטימיזציות וראיות.
- סכמת Supabase ומיגרציה בטוחה למערכת קיימת.
- גיבוי ושחזור של כל ישויות v1.0.

## אמינות

- Q1, Q2 ו־Q3 האמיתיים נבדקו בנפרד.
- בכל קובץ זוהו 4 אזורים ו־12 שורות פעילות.
- מחקרי השוק שבתוך החוברות נקלטו אוטומטית.
- לא נמצאו שגיאות חילוץ או הפרות אימות בדוחות שנבדקו.
- מערך הבדיקות: 70 עברו; 12 בדיקות אינטגרציה דורשות Supabase חי ולכן דולגו מקומית.

## מגבלה מכוונת

המערכת אינה שולחת החלטות ל־INTOPIA, אינה מאשרת דוח בעצמה ואינה משנה Actuals או חוק קשיח ללא אישור אנושי.

## תלויות ואופטימיזציה משולבת

- חדר ההחלטות מציג כעת מהלך החלטות מתואם ולא רק המלצות נפרדות.
- המנוע מזהה תנאים מקדימים, פעולות שיש לקבוע יחד, פעולות שנפתחות בזכות החלטה אחרת והתנגשויות בין החלטות.
- התלויות כוללות בין היתר מחקר לפני השקעה בלתי הפיכה, אספקת X לפני ייצור Y, קיבולת לפני הגדלת ייצור, תיאום מחיר–פרסום–מלאי ומימון לפני סל פעולות החורג מהתקציב.
- לכל המלצה מוצגים השלב שלה במהלך הכולל, התלויות שלה והמידע או הפעולה החסרים.
- מעבדת התרחישים מחזירה סדר ביצוע, מסלול קריטי, עלות משולבת ותקציב אפקטיבי לאחר פעולות מימון.
- ה־Decision Agent הונחה לנתח תיק החלטות שלם, לזהות תלות בתקציב ובתזמון, ולהסביר את ההשפעה המשותפת עד Q9.
